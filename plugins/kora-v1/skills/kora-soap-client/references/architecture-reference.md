```java
            HttpClient httpClient,
```

## Runtime Execution Flow

```
User Code: soapClient.test(request)
              │
              ▼
┌─────────────────────────────────────────┐
│ SimpleServiceImpl.test()                │
│  1. Create SOAP envelope (XML)          │
│  2. Start telemetry span                │
│  3. Marshal request to bytes            │
└─────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│ SoapRequestExecutor                     │
│  4. Log request (if enabled)            │
│  5. Record metrics start                │
│  6. HTTP POST to config.url             │
└─────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│ HTTP Client (JDK/Apache/Undertow)       │
│  7. TCP connection                      │
│  8. Send HTTP request with SOAP body    │
│  9. Receive HTTP response               │
└─────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│ SoapRequestExecutor                     │
│  10. Parse SOAP response                │

              │
              ▼
```

## Reactive Methods

All generated `*Impl` classes include reactive variants:

```java
public class SimpleServiceImpl implements SimpleService, SoapClient {

    // Blocking (from interface)
    @Override
    public TestResponse test(TestRequest request) {
        return blockingCall(request);
    }

    // Reactive (Kora extension)
    public Mono<TestResponse> testReactive(TestRequest request) {
        return Mono.fromCallable(() -> test(request))
            .subscribeOn(Schedulers.boundedElastic());
    }

    // All methods have reactive variants
    public Mono<GetInfoResponse> getInfoReactive(String id) {
        return Mono.fromCallable(() -> getInfo(id));
    }
}
```

**Usage:**
```java
@Component
public class AsyncService {
    private final SimpleServiceImpl client;

    public AsyncService(SimpleServiceImpl client) {
        this.client = client;
    }

    public Mono<Void> processAll() {
        return Flux.just("id1", "id2", "id3")
            .flatMap(client::getInfoReactive)
            .then();
    }
}
```

## Thread Safety

**Generated clients are thread-safe:**

- `SimpleServiceImpl` is stateless
- `HttpClient` is thread-safe (connection pooled)
- `XmlTools` uses thread-local parsers
- Safe for singleton injection

```java
@Component  // Singleton by default
public final class MyService {

    private final SimpleService soapClient;  // Thread-safe

    public MyService(SimpleService soapClient) {
        this.soapClient = soapClient;
    }

    // Safe for concurrent requests
    public void handleRequest(String id) {
        soapClient.test(new TestRequest(id));
    }
}
```

## Performance Characteristics

| Operation | Latency | Notes |
|-----------|---------|-------|
| SOAP call (network) | 10-500ms | Depends on service |
| XML marshalling | 0.1-2ms | JAXB under the hood |
| XML unmarshalling | 0.1-2ms | JAXB under the hood |
| Telemetry overhead | <0.5ms | When enabled |
| Logging overhead | 1-5ms | When enabled, sync |

**Optimization tips:**
1. Use reactive methods for high concurrency
2. Disable logging in production (use metrics/tracing instead)
3. Configure connection pool in HTTP client
4. Set appropriate timeouts per service
