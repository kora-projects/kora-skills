Declarative method logging is provided by the `logging-common` module (transitively present via
`logging-logback`). The aspects are generated by the Kora annotation processor, so it must be on
the build (`annotationProcessor "ru.tinkoff.kora:annotation-processors"` / Kotlin
`ksp "ru.tinkoff.kora:symbol-processors"`). The annotated class must be non-`final` (Java) or
`open` (Kotlin) for aspects to apply.

## Dependency

Usually already present via `logging-logback`; otherwise add it explicitly and extend the module.

```groovy
implementation "ru.tinkoff.kora:logging-common"
```

```java
@KoraApp
public interface Application extends LoggingModule { }
```

## @Log ŌĆö argument and result logging

`@Log` logs both arguments (on entry, `>`) and the result (on exit, `<`). Use `@Log.in` for
arguments only and `@Log.out` for the result only. At `INFO` only the markers `>`/`<` are logged;
at `DEBUG`/`TRACE` the data block is included.

```java
import ru.tinkoff.kora.logging.common.annotation.Log;

@Log
public String methodWithArgs(String strParam, int numParam) {
    return "testResult";
}
// DEBUG r.t.e.Example.methodWithArgs: > {data: {strParam: "s", numParam: "4"}}
// DEBUG r.t.e.Example.methodWithArgs: < {data: {out: "testResult"}}
```

```java
@Log.in
public String onlyArgs(String strParam, int numParam) { return "r"; }

@Log.out
public String onlyResult(String strParam, int numParam) { return "r"; }
```

## @Log.off ŌĆö exclude data

`@Log.off` on a parameter excludes that argument from the logged data (e.g. secrets). On a method
combined with `@Log.out` it suppresses the argument block while keeping the result marker.

```java
@Log.in
public String login(String user, @Log.off String password) {
    return "token";
}
// password is omitted from the logged arguments
```

## @Mdc ŌĆö enrich MDC

`@Mdc` adds key/value pairs to MDC for the duration of the method. It applies to methods and
parameters and is repeatable.

- `key()` ŌĆö MDC key; defaults to the parameter name.
- `value()` ŌĆö MDC value; defaults to the parameter value. Supports `${...}` generated expressions.
- `global()` ŌĆö if `true`, the value stays in the thread MDC after the method returns.

```java
import ru.tinkoff.kora.logging.common.annotation.Mdc;

// Parameter: key defaults to the parameter name "orderId", value is the argument
public String byParam(@Mdc String orderId) { return "1"; }

// Parameter with explicit key
public String byKey(@Mdc(key = "order") String orderId) { return "1"; }

// Method-level static value
@Mdc(key = "stage", value = "checkout")
public String fixed(String s) { return "1"; }

// Generated value
@Mdc(key = "traceId", value = "${java.util.UUID.randomUUID().toString()}")
public String generated(String s) { return "1"; }

// Combined: two method annotations plus one parameter annotation
@Mdc(key = "traceId", value = "${java.util.UUID.randomUUID().toString()}", global = true)
@Mdc(key = "stage", value = "checkout")
public Integer combined(@Mdc(key = "order") String orderId) { return 1; }
```

Example log line with MDC:

```
INFO [main] r.t.e.Example.test: > {data: {s: "testValue"}} traceId=some-uuid stage=checkout order=testValue
```

## Supported signatures

===! "Java"

    Class must be non-`final`. `T` is the return type, or `Void`.

    - `T myMethod()`
    - `Optional<T> myMethod()`
    - `CompletionStage<T> myMethod()`
    - `Mono<T> myMethod()` / `Flux<T> myMethod()` (require `io.projectreactor:reactor-core`)

=== "Kotlin"

    Class must be `open`. `T` is the return type, `T?`, or `Unit`.

    - `fun myMethod(): T`
    - `suspend fun myMethod(): T` (require `org.jetbrains.kotlinx:kotlinx-coroutines-core`)
    - `fun myMethod(): Flow<T>` (require `org.jetbrains.kotlinx:kotlinx-coroutines-core`)
